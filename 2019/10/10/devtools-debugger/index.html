Chrome Devtools Debugger

发表于 2019-10-10 分类于 chrome devtools protocol

调试器域公开了JavaScript调试功能。 它允许设置和删除断点，逐步执行，探索堆栈跟踪等。

Methods

Debugger.continueToLocation

继续执行直到到达特定位置

参数

  • location Location 位置继续
  • targetCallFrames[optional] string 允许值: any, current

Debugger.disable

禁用给定页面的调试器

Debugger.enable

为给定的页面启用调试器。 在收到此命令的结果之前，客户端不应假定调试已启用。

参数

  • maxScriptsCacheSize[optional] number 调试器可以容纳的已收集脚本（其他堆对象未引用）的最大大小（以字节为单位）。 如果省略参数，则没有限制。[实验性质]

返回

  • debuggerId Runtime.UniqueDebuggerId 调试器的唯一标识符。[实验性质]

Debugger.evaluateOnCallFrame

在给定的调用框架上评估表达式。

参数

  • callFrameId CallFrameId 框架标识符
  • expression string 要评估的表达式
  • objectGroup[optional] string 要放入结果的字符串对象组名称（允许使用releaseObjectGroup快速释放结果对象句柄）。
  • includeCommandLineAPI[optional] boolean 指定命令行API是否应可用于求值表达式，默认为false
  • silent[optional] boolean 在静默模式下，不会报告评估期间抛出的异常，并且不会暂停执行。 覆盖setPauseOnException状态。
  • returnByValue[optional] boolean 结果是否应为应按值发送的JSON对象
  • generatePreview[optional] boolean 是否应为结果生成预览。 [实验性质]
  • throwOnSideEffect[optional] boolean 如果在评估过程中不能排除副作用，是否抛出异常。
  • timeout[optional] Runtime.TimeDelta 超时后终止执行（毫秒数）[实验性质]

返回

  • result Runtime.RemoteObject 评估结果的对象包装
  • exceptionDetails[optional] Runtime.ExceptionDetails 异常详细信息

Debugger.getPossibleBreakpoints

返回断点的可能位置。 开始和结束范围位置中的scriptId应该相同。

参数

  • start Location 范围的起点，以搜索可能的断点位置。
  • end[optional] Location 在（不包括）中搜索可能的断点位置的范围的结尾。 如果未指定，则将脚本结尾用作范围结尾。
  • restrictToFunction[optional] boolean 仅考虑与开始具有相同（非嵌套）功能的位置。

返回

  • locations array BreakLocation 可能的断点位置列表。

Debugger.getScriptSource

返回具有给定id的脚本的源。

参数

  • scriptId Runtime.ScriptId 要获取源代码的脚本的ID。

返回

  • scriptSource string 脚本源代码

Debugger.getWasmBytecode

返回具有给定ID的WebAssembly脚本的字节码。

参数

  • scriptId Runtime.ScriptId Wasm脚本的ID以获取源代码。

返回

  • bytecode string

Debugger.getStackTrace

返回具有给定stackTraceId的堆栈跟踪[实验性质]

参数

  • stackTraceId Runtime.StackTraceId

返回

  • stackTrace Runtime.StackTrace

Debugger.pause

在下一个JavaScript语句上停止。

Debugger.pauseOnAsyncCall

禁用给定页面的调试器[实验性质][已淘汰]

参数

  • parentStackTraceId Runtime.StackTraceId 当启动具有给定堆栈跟踪的异步调用时，调试器将暂停

Debugger.removeBreakpoint

删除JavaScript断点

参数

  • breakpointId BreakpointId

Debugger.restartFrame

从头开始重新启动特定的呼叫帧。

参数

  • callFrameId CallFrameId 调用框架标识符以进行评估

返回

  • callFrames array CallFrame 新的堆栈跟踪
  • asyncStackTrace[optional] Runtime.StackTrace 异步堆栈跟踪（如果有）。
  • asyncStackTraceId[optional] Runtime.StackTraceId 异步堆栈跟踪（如果有）。[实验性质]

Debugger.resume

恢复JavaScript执行

Debugger.searchInContent

在脚本内容中搜索给定的字符串

参数

  • scriptId Runtime.ScriptId 要搜索的脚本的ID
  • query string 要搜索的字符串
  • caseSensitive[optional] boolean 如果为true，则搜索区分大小写。
  • isRegex[optional] boolean 如果为true，则将字符串参数视为正则表达式。

返回

  • result array SearchMatch 搜索匹配列表

Debugger.setAsyncCallStackDepth

启用或禁用异步调用堆栈跟踪

参数

  • maxDepth integer 异步调用堆栈的最大深度。 设置为0将有效地禁用收集异步调用堆栈（默认）。

Debugger.setBlackboxPatterns

用传递的模式替换以前的黑盒模式。 强制后端在网址与模式之一匹配的脚本中跳过步进/暂停。 虚拟机将尝试执行多次“介入”来离开黑匣子脚本，如果操作不成功则最终求助于“脱离”[实验性质]

参数

  • patterns array [string] 正则表达式数组，将用于检查脚本URL的黑盒状态。

Debugger.setBlackboxedRanges

在黑框范围内在脚本中进行后端跳过步骤。 VM将尝试执行多次“进入”操作，从而将被列入黑名单的脚本尝试，如果未成功，则最终求助于“退出”。 Positions数组包含更改黑盒状态的位置。 第一个时间间隔未加黑框。 数组应排序。[实验性质]

参数

  • scriptId Runtime.ScriptId
  • positions array ScriptPosition

Debugger.setBreakpoint

在给定位置设置JavaScript断点

参数

  • location Location 设置断点的位置
  • condition[optional] string 用作断点条件的表达式。 如果指定，调试器将仅在此表达式的值为true时才在断点处停止

返回

  • breakpointId BreakpointId 创建的断点的ID，以供进一步参考。
  • actualLocation Location 此断点解析到的位置

Debugger.setInstrumentationBreakpoint

设置检测断点

参数

  • instrumentation string 仪器名称 允许值:beforeScriptExecution, beforeScriptWithSourceMapExecution

返回

  • breakpointId BreakpointId 创建的断点的ID，以供进一步参考

Debugger.setBreakpointByUrl

在URL或URL正则表达式指定的给定位置设置JavaScript断点。 发出此命令后，所有现有的已解析脚本都将解析断点并在locations属性中返回。 进一步的匹配脚本分析将导致后续的breakpointResolved事件发出。 此逻辑断点将在页面重新加载后继续存在。

参数

  • lineNumber integer 设置断点的行号
  • url[optional] string 设置断点的资源的URL
  • urlRegex[optional] string 用于设置断点的资源URL的正则表达式模式。 必须指定url或urlRegex
  • scriptHash[optional] string 用于设置断点的资源的脚本哈希
  • columnNumber[optional] integer 在行中设置断点的偏移量
  • condition[optional] string 用作断点条件的表达式。 如果指定，调试器将仅在此表达式的值为true时才在断点处停止

返回

  • breakpointId BreakpointId 创建的断点的ID，以供进一步参考
  • locations array Location 添加后此断点解析为的位置的列表

Debugger.setBreakpointOnFunctionCall

在每次调用给定函数之前设置JavaScript断点。 如果另一个函数是从与给定源相同的源创建的，则调用该函数也会触发断点[实验性质]

参数

  • objectId Runtime.RemoteObjectId 功能对象ID
  • condition[optional] string 用作断点条件的表达式。 指定后，如果此表达式的值为true，调试器将在断点处停止

返回

  • breakpointId BreakpointId 创建的断点的ID供进一步参考

Debugger.setBreakpointsActive

激活/禁用页面上的所有断点

参数

  • active boolean 断点活动状态的新值

Debugger.setPauseOnExceptions

定义异常状态的暂停。 可以设置为在所有异常，未捕获的异常或无异常时停止。 异常状态的初始暂停为无

参数

  • state string 在例外模式下暂停 允许值:none, uncaught, all

Debugger.setReturnValue

更改顶框中的返回值。 仅在返回中断位置可用。[实验性质]

参数

  • newValue Runtime.CallArgument 新的返回值

Debugger.setScriptSource

实时编辑JavaScript源

参数

  • scriptId Runtime.ScriptId 要编辑的脚本的ID
  • scriptSource string 脚本的新内容
  • dryRun[optional] boolean 如果为true，则更改实际上不会应用。 空运行可用于获取结果描述，而无需实际修改代码

返回

  • callFrames[optional] array CallFrame 停止VM时进行编辑时会出现新的堆栈跟踪
  • stackChanged[optional] boolean 应用更改后是否修改了当前调用堆栈
  • asyncStackTrace[optional] Runtime.StackTrace 异步堆栈跟踪（如果有）
  • asyncStackTraceId[optional] Runtime.StackTraceId 异步堆栈跟踪（如果有）[实验性质]
  • exceptionDetails[optional] Runtime.ExceptionDetails异常详细信息（如果有）

Debugger.setSkipAllPauses

使页面不因任何暂停而中断（断点，异常，dom异常等）。

参数

  • skip boolean 跳过暂停状态的新值

Debugger.setVariableValue

更改调用框中变量的值。 不支持基于对象的作用域，必须手动对其进行更改。

参数

  • scopeNumber integer 范围链中列出的基于0的范围数。 仅允许使用“本地”，“关闭”和“捕获”范围类型。 其他范围可以手动操作。
  • variableName string 变量名。
  • newValue Runtime.CallArgument 新的变量值。
  • callFrameId CallFrameId 包含变量的调用框架的ID。

Debugger.stepInto

进入函数调用。

参数

  • breakOnAsyncCall[optional] boolean 调试器将在执行下一个暂停之前计划的第一个异步任务时暂停。[实验性质]

Debugger.stepOut

退出函数调用。

Debugger.stepOver

逐步执行该语句。

Events

Debugger.breakpointResolved

将断点解析为实际脚本和位置时触发。

参数

  • breakpointId BreakpointId 断点唯一标识符
  • location Location 实际断点位置

Debugger.paused

当虚拟机在断点或异常或任何其他停止条件上停止时触发。

参数

  • callFrames array CallFrame 虚拟机停止在其上的调用堆栈
  • reason string 暂停原因 允许值: ambiguous, assert, debugCommand, DOM, EventListener, exception, instrumentation, OOM, other, promiseRejection, XHR
  • data[optional] object 包含特定于中断的辅助属性的对象
  • hitBreakpoints[optional] array [ string ] 命中断点ID
  • asyncStackTrace[optional] Runtime.StackTrace 异步堆栈跟踪（如果有）。
  • asyncStackTraceId[optional] Runtime.StackTraceId 异步堆栈跟踪（如果有）。[实验性质]
  • asyncCallStackTraceId[optional] Runtime.StackTraceId 从不存在，将被删除。[实验性质][已淘汰]

Debugger.resumed

虚拟机恢复执行时触发

Debugger.scriptFailedToParse

当虚拟机无法解析脚本时触发

参数

  • scriptId Runtime.ScriptId 解析脚本的标识符
  • url string 解析的脚本的URL或名称（如果有）
  • startLine integer 具有给定URL的资源内脚本的行偏移量（用于脚本标签）
  • startColumn integer 具有给定URL的资源中脚本的列偏移量
  • endLine integer 脚本的最后一行
  • endColumn integer 脚本最后一行的长度
  • executionContextId Runtime.ExecutionContextId 指定脚本创建上下文
  • hash string 脚本的内容哈希
  • executionContextAuxData[optional] object 嵌入程序专用辅助数据
  • sourceMapURL[optional] string 与脚本关联的源映射的URL（如果有）。
  • hasSourceURL[optional] boolean 如果此脚本具有sourceURL，则为True。
  • isModule[optional] boolean 如果该脚本是ES6模块，则为true。
  • length[optional] integer 此脚本的长度。
  • stackTrace[optional] Runtime.StackTrace 脚本解析事件触发位置的JavaScript顶部堆栈框架（如果有）。[实验性质]

Debugger.scriptParsed

虚拟机解析脚本时触发。 启用调试器后，也会针对所有已知和未收集的脚本触发此事件。

参数

  • scriptId Runtime.ScriptId 解析脚本的标识符
  • url string 解析的脚本的URL或名称（如果有）
  • startLine integer 具有给定URL的资源内脚本的行偏移量（用于脚本标签）
  • startColumn integer 具有给定URL的资源中脚本的列偏移量
  • endLine integer 脚本的最后一行
  • endColumn integer 脚本最后一行的长度
  • executionContextId Runtime.ExecutionContextId 指定脚本创建上下文
  • hash string 脚本的内容哈希
  • executionContextAuxData[optional] object 嵌入程序专用辅助数据
  • isLiveEdit[optional] boolean 是的，如果此脚本是通过实时编辑操作生成的 [实验性质]
  • sourceMapURL[optional] string 与脚本关联的源映射的URL（如果有）。
  • hasSourceURL[optional] boolean 如果此脚本具有sourceURL，则为True。
  • isModule[optional] boolean 如果该脚本是ES6模块，则为true。
  • length[optional] integer 此脚本的长度。
  • stackTrace[optional] Runtime.StackTrace 脚本解析事件触发位置的JavaScript顶部堆栈框架（如果有）[实验性质]

Types

BreakpointId

断点标识符

Type: string

CallFrameId

呼叫帧标识符

Type: string

Location

在源代码中的位置

Type: Object

数据结构

  • scriptId Runtime.ScriptId 在Debugger.scriptParsed中报告的脚本标识符。
  • lineNumber integer 脚本中的行号（从0开始）
  • columnNumber[optional] integer 脚本中的列号（从0开始）

ScriptPosition

源代码中的位置[实验性质]

Type: Object

数据结构

  • lineNumber integer
  • columnNumber integer

CallFrame

JavaScript调用框架。 调用帧数组构成调用堆栈

Type: Object

数据结构

  • callFrameId CallFrameId 呼叫帧标识符。 该标识符仅在虚拟机暂停时有效
  • functionName string 在此调用框架上调用的JavaScript函数的名称
  • functionLocation[optional] Location 源代码中的位置
  • location Location 源代码中的位置
  • url string JavaScript脚本名称或网址
  • scopeChain array Scope 此调用框架的范围链
  • this Runtime.RemoteObject 此调用框架的此对象
  • returnValue Runtime.RemoteObject 如果函数在返回点，则返回值

Scope

范围说明

Type: Object

数据结构

  • type string 范围说明 允许值: global, local, with, closure, catch, block, script, eval, module
  • object Runtime.RemoteObject 表示范围的对象。 对于全局和范围，它表示实际对象； 对于其余的范围，它是人为的瞬时对象，将范围变量枚举为其属性。
  • name[optional] string
  • startLocation[optional] Location 源代码中范围开始的位置
  • endLocation[optional] Location 源代码中作用域结束的位置

SearchMatch

搜索资源匹配项

Type: Object

数据结构

  • lineNumber number 资源内容中的行号
  • lineContent string 符合内容的行

BreakLocation

Type: Object

数据结构

  • scriptId Runtime.ScriptId 在Debugger.scriptParsed中报告的脚本标识符
  • lineNumber number 脚本中的行号（从0开始）
  • columnNumber[optional] string 脚本中的列号（从0开始）
  • type[optional] string 范围说明 允许值: debuggerStatement, call, return
